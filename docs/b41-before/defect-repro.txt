D1: Apostrophe word boundary breaks Speak highlight (RECURRING)
============================================================
Title containing apostrophe ("Cole's Big Show") + body containing possessives + contractions
causes the highlight to fall one word behind. Reported across multiple sessions; previous
fix attempts regressed.

Investigation findings (b41):
- wrapStoryWords (index.html:3940) splits text-node content on /(\s+)/ — preserves apostrophes
- buildAlignmentTimings (index.html:3998) uses /\S+/g — preserves apostrophes
- Both filter pure-punctuation tokens via [a-zA-Z0-9] test
- Both *should* produce 1:1 token counts
- The defect notes the fix keeps regressing — no single source of truth for "what is a word
  in a NoddyTales story?". Two parallel implementations diverge under future edits.

b41 architectural fix: extract shared tokenizeStoryText(s) → returns array of word-with-
punctuation tokens. Both wrap + alignment paths consume it. Unit test covers possessives,
contractions, hyphens, leading/trailing punct. Document as recurring class in CLAUDE.md.

      - engine-v2.js:4051 v3_ls_payoff_chant_suspect_caves

      - engine-v2.js:4621 v3_rl_payoff_tool_chant
      - engine-v2.js:5450 FLAVOR_CALLBACKS.payoff_word "The backpack said..."
      - And ~3 more.

      applauded the wrong order. The show was now better." (engine-v2.js:4457)
      — visual gag without a comedic direction. Prop fixes itself "in roughly the wrong

  2e. "Somebody had brought hot dogs. Nobody knew when. Nobody minded." — comes from a

  2f. Story length: 5 paragraphs but P4 is bloated by callbacks (~10 sentences) — total

D3: Bedtime mode selected but anytime ending fired
============================================================
The reported ending — "Backstage, Cole high-fived the lynx. The show was unrepeatable,
which felt right. Time to pack up and find the next thing." — is verbatim
v3_sw_landing_any at engine-v2.js:4487, which is tagged mode:'anytime'.

This beat ONLY fires when picks.storyMode === 'anytime' (engine-v2.js:5242). For the user
to receive this ending despite "bedtime selected", picks.storyMode must have been 'anytime'
at story-build time. Likely causes:
  a) Stale Profile.storyMode from a prior session (nt_story_mode='anytime' persisted)
  b) The storyMode picker step was not surfaced for the returning user, so the previously-
     persisted value silently applied
  c) UI tile selection didn't update state.storyMode (timing race / event-handler bug)

Engineering response: the architecture must make storyMode determinism testable and
visible. b41 adds:
  - Audit/tag every V3 landing beat with explicit mode:'bedtime' or mode:'anytime'
  - Rewrite untagged landings whose content isn't actually bedtime to include bedtime
    closure (yawning, going to sleep, tucked in)
  - QA Section 21 gate: generate 60 stories per kid/big/tween with picks.storyMode=
    'bedtime' → final paragraph must contain bedtime lexicon; 0 leaks across all 180.

D4: Secondary character (sidekick) added but never appeared in story
============================================================
Engine flow:
  - state.sidekicks (Profile, persisted) → buildStory passes through → generateStoryV3
  - engine-v2.js:3091-3092: picks one name from state.sidekicks if non-empty, else null
  - Renders as slots.sidekick = { name, cap, lc } only when present
  - V3 beats use {sidekick.cap}/{sidekick.lc} tokens — but only ONE V3 beat
    (V2 only: h3 at engine-v2.js:1726) actually references sidekick

V3 kid/big/tween blueprints reference 'ally' (companion pick) extensively, not 'sidekick'.
So when state.sidekicks has a name (e.g., "Riley"), the engine picks Riley as sidekickName
but no V3 beat uses {sidekick.*} tokens, and the rendered story never mentions Riley.

Fix: when state.sidekicks is non-empty AND the rendered story body does not contain the
sidekick name, append a 1-sentence sidekick cameo to the landing paragraph. New
ensureSidekickAppears post-pass in generateStoryV3.

Section 21 gate: state.sidekicks=['Riley'] → rendered body contains 'Riley' across all
4 V3 blueprints × 3 tiers × 20 reps = 240 stories. 0 leaks.
